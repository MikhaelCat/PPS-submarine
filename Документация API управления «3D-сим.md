# **Документация API управления «3D-симулятор АНПА»**

Данный API предназначен для взаимодействия с симулятором через протокол UDP. Обмен данными происходит в формате JSON.

**Общие параметры сети**

&#x20;\* Протокол: UDP

&#x20;\* Порт управления (Requests): 8080 (по умолчанию) — для отправки команд и получения ответов.

&#x20;\* Порт телеметрии (Stream): 8081 — односторонняя рассылка данных от симулятора (стрим).

&#x20;\* Кодировка: UTF-8

**1. Формат сообщений**

Запрос (Client -> Unity)

Каждое сообщение должно быть валидным JSON-объектом:

{

&#x20; "request": "название\_команды",

&#x20; "values": {

&#x20;   "параметр1": "значение",

&#x20;   "параметр2": 123

&#x20; },

&#x20; "request\_id": 1648821000

}



&#x20;\* request: Строковый идентификатор команды.

&#x20;\* values: Словарь параметров (может быть пустым {}).

&#x20;\* request\_id: Произвольное целое число для сопоставления ответа с запросом.

Ответ (Unity -> Client)

Симулятор возвращает ответ на каждый запрос на тот же адрес/порт, с которого пришла команда:

{

&#x20; "status": 200,

&#x20; "message": "Success",

&#x20; "result": { ... }

}



&#x20;\* Статусы: 200 (ОК), 400 (Ошибка параметров), 404 (Команда не найдена), 500 (Внутренняя ошибка симулятора).

**2. Список доступных команд**

set\_motor\_speed

Устанавливает тягу для конкретного мотора указанного аппарата.

&#x20;\* Параметры (values):

&#x20;  \* auv\_id (int): ID аппарата.

&#x20;  \* motor\_id (int): ID мотора (обычно 1-4).

&#x20;  \* force (float): Значение тяги (обычно от -1.0 до 1.0).

&#x20;\* Результат: {"status": 0} (где 0 — код успеха внутреннего контроллера).

reset\_auv

Мгновенно перемещает аппарат в начальную точку и сбрасывает его физические показатели (скорость, вращение).

&#x20;\* Параметры (values):

&#x20;  \* auv\_id (int): ID аппарата для сброса.

&#x20;\* Действие: Отключает физику, сбрасывает линейную и угловую скорости в 0, выставляет стартовую позицию, обнуляет тягу всех моторов и включает физику обратно.

&#x20;\* Результат: {"reset\_id": auvId}.

**3. Формат стриминга данных (Порт 8081)**

Симулятор автоматически рассылает JSON-пакеты с частотой обновления сенсоров (в текущей реализации \~2.5 Гц для сенсоров и чаще для списка ID).

Тип: auv\_list

Рассылается раз в секунду. Содержит список всех активных ID аппаратов на сцене.

{

&#x20; "type": "auv\_list",

&#x20; "ids": \[0, 1, 2]

}



Тип: telemetry

Основные навигационные данные.

&#x20;\* position: {x, y, z}

&#x20;\* rotation: {x, y, z} (Эйлеровы углы)

&#x20;\* depth: (float) Глубина.

Тип: mbes (Многолучевой эхолот)

&#x20;\* points\_x, points\_y: Массивы координат точек сканирования дна.

Тип: sonar (ГБО)

&#x20;\* left, right: Массивы интенсивностей отраженного сигнала.

&#x20;\* max\_range: Максимальная дистанция обзора.

Тип: camera

&#x20;\* camera\_image: Строка в формате Base64 (изображение в формате JPG).

**4. Обработка ошибок**

Если в ServerLauncher происходит исключение (например, неверный формат данных в values), API вернет:

&#x20;\* status: 400

&#x20;\* message: Текст ошибки (например, "Input string was not in a correct format").
